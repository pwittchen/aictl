AICTL · v0.36.0 · Rust · HTTP LLM proxy · OpenAI-compatible

server

An HTTP LLM proxy with built-in redaction, prompt-injection blocking, logs, and a per-process audit trail. Point any OpenAI SDK at it to call Anthropic, Gemini, Grok, Mistral, DeepSeek, Kimi, Z.ai, Ollama, GGUF, or MLX models — one endpoint, one master key.

$ curl -fsSL https://aictl.app/server/install.sh | sh

// install

One liner

macOS arm64/x86_64 and Linux x86_64/arm64 ship as prebuilt binaries. The installer detects your platform, downloads the matching artifact from the latest GitHub release, and drops it on $PATH. Unsupported platforms fall back to cargo install.

$ curl -fsSL https://aictl.app/server/install.sh | sh

First launch

On first launch the server generates a 32-byte master API key, persists it to ~/.aictl/config as AICTL_SERVER_MASTER_KEY, and prints it once to stderr. Copy it — every authenticated request needs Authorization: Bearer <key>.

Shared config

The server reads the same ~/.aictl/config the CLI reads. Provider keys, redaction rules, and the prompt-injection guard are tuned once and apply to both surfaces.

// quickstart

Send your first request

Start the server, point an OpenAI SDK at it, send a request. The model id picks the upstream provider — Anthropic, Gemini, Grok, Mistral, DeepSeek, Kimi, Z.ai, Ollama, GGUF, or MLX.

$ aictl-server

If you have the CLI installed, aictl --serve is a convenience shortcut that locates the sibling aictl-server binary and runs it. Trailing args forward verbatim, e.g. aictl --serve -- --bind 0.0.0.0:7878 --quiet.

# Python — point the OpenAI SDK at the proxy. from openai import OpenAI client = OpenAI( api_key="<AICTL_SERVER_MASTER_KEY>", base_url="http://127.0.0.1:7878/v1", ) reply = client.chat.completions.create( model="claude-sonnet-4-6", messages=[{"role": "user", "content": "Hello"}], ) print(reply.choices[0].message.content)
# Plain curl — same endpoint, same master key, any provider. curl http://127.0.0.1:7878/v1/chat/completions \ -H "Authorization: Bearer $AICTL_SERVER_MASTER_KEY" \ -H "Content-Type: application/json" \ -d '{ "model": "gpt-4o-mini", "messages": [{"role": "user", "content": "Hello"}] }'
// rest api

Endpoints

Five endpoints. Every authenticated request needs
Authorization: Bearer <master-key>.
GET /healthz is the only auth-free route.

POST /v1/chat/completions auth required

OpenAI-shaped chat endpoint. The model field picks the provider. stream: true returns SSE in OpenAI's data: {"choices":[{"delta":...}]} shape, ending with data: [DONE].

Request
curl -X POST http://127.0.0.1:7878/v1/chat/completions \ -H "Authorization: Bearer $AICTL_SERVER_MASTER_KEY" \ -H "Content-Type: application/json" \ -d '{ "model": "claude-sonnet-4-6", "messages": [{"role": "user", "content": "Hello"}] }'
Response
{ "id": "chatcmpl-7f3a92...", "object": "chat.completion", "created": 1714411200, "model": "claude-sonnet-4-6", "choices": [{ "index": 0, "message": {"role": "assistant", "content": "Hi!"}, "finish_reason": "stop" }], "usage": {"prompt_tokens": 8, "completion_tokens": 2, "total_tokens": 10} }
POST /v1/completions auth required

Legacy text-completion API. The prompt is wrapped in a single user message and routed through the same provider selection logic.

Request
curl -X POST http://127.0.0.1:7878/v1/completions \ -H "Authorization: Bearer $AICTL_SERVER_MASTER_KEY" \ -H "Content-Type: application/json" \ -d '{ "model": "gpt-4o-mini", "prompt": "Once upon a time" }'
Response
{ "id": "cmpl-3a91...", "object": "text_completion", "created": 1714411200, "model": "gpt-4o-mini", "choices": [{ "index": 0, "text": " in a faraway land...", "finish_reason": "stop" }], "usage": {"prompt_tokens": 5, "completion_tokens": 20, "total_tokens": 25} }
GET /v1/models auth required

Lists every model in the catalogue plus locally available Ollama / GGUF / MLX models, with context_window and an available flag (true when the upstream key is configured).

Request
curl http://127.0.0.1:7878/v1/models \ -H "Authorization: Bearer $AICTL_SERVER_MASTER_KEY"
Response
{ "object": "list", "data": [ { "id": "claude-sonnet-4-6", "object": "model", "owned_by": "Anthropic", "context_window": 200000, "available": true }, { "id": "gpt-4o-mini", "object": "model", "owned_by": "OpenAI", "context_window": 128000, "available": true } ] }
GET /v1/stats auth required

Returns the same usage aggregates the CLI's /stats command shows — today, this month, overall.

Request
curl http://127.0.0.1:7878/v1/stats \ -H "Authorization: Bearer $AICTL_SERVER_MASTER_KEY"
Response
{ "today": { "sessions": 0, "requests": 12, "llm_calls": 12, "tool_calls": 0, "input_tokens": 1840, "output_tokens": 412, "cost_usd": 0.018, "models": {"claude-sonnet-4-6": 8, "gpt-4o-mini": 4} }, "this_month": { /* same shape */ }, "overall": { /* same shape */ } }
GET /healthz public

Liveness probe. Returns status, version, uptime_secs, and active_requests.

Request
curl http://127.0.0.1:7878/healthz
Response
{ "status": "ok", "version": "0.36.0", "uptime_secs": 3812, "active_requests": 0 }

Every error response is {"error":{"code":"…","message":"…"}} — the same envelope OpenAI uses, so existing SDK error handlers keep working unchanged.

// configuration

Knobs

Every server-only key lives in ~/.aictl/config with the AICTL_SERVER_* prefix. CLI flags override config; config overrides defaults.

AICTL_SERVER_BIND=127.0.0.1:7878 # listen address AICTL_SERVER_MASTER_KEY=<auto-generated> # bearer token AICTL_SERVER_REQUEST_TIMEOUT=120 # per-request seconds; 0 disables AICTL_SERVER_BODY_LIMIT_BYTES=2097152 # 2 MiB AICTL_SERVER_MAX_CONCURRENT_REQUESTS=32 # 503 when saturated AICTL_SERVER_SHUTDOWN_TIMEOUT=20 # SIGTERM drain seconds AICTL_SERVER_SSE_KEEPALIVE=15 # 0 disables AICTL_SERVER_LOG_LEVEL=info # trace|debug|info|warn|error AICTL_SERVER_LOG_FILE=~/.aictl/server.log # empty disables file sink AICTL_SERVER_LOG_BODIES=true # log redacted bodies AICTL_SERVER_CORS_ORIGINS= # empty = CORS off AICTL_SERVER_RATE_LIMIT_RPM=0 # per-IP req/min; 0 disables AICTL_SERVER_RATE_LIMIT_BURST=0 # bucket capacity; 0 = RPM

Server-scoped security flags. Each takes precedence over the matching AICTL_* form when the engine is loaded inside aictl-server; unset overrides fall through to the shared key, so a single-host setup needs no duplication. Tool-dispatch knobs (CWD jail, shell allow/block lists, blocked env vars, disabled tools, max-write byte cap, shell timeout) are intentionally not mirrored — the server does not run tools.

AICTL_SERVER_SECURITY=true # master switch (injection guard + audit); false/0 disables AICTL_SERVER_SECURITY_INJECTION_GUARD=true # run detect_prompt_injection on every user message AICTL_SERVER_SECURITY_AUDIT_LOG=true # gateway:<provider> + redaction events to the audit log AICTL_SERVER_SECURITY_REDACTION=off # off|redact|block (block → 400 redaction_blocked) AICTL_SERVER_SECURITY_REDACTION_LOCAL=false # also redact for local providers (Ollama/GGUF/MLX)

Server-scoped redaction tuning. Same precedence rule as the security flags above — unset server overrides fall through to the matching AICTL_REDACTION_* key. Detector subsetting, custom patterns, allowlists, and the optional Layer-C NER model can be tuned independently for the proxy without forking the CLI's config.

AICTL_SERVER_REDACTION_DETECTORS= # empty = all; subset of api_key,aws,jwt,private_key,connection_string,credit_card,iban,email,phone,high_entropy AICTL_SERVER_REDACTION_EXTRA_PATTERNS= # semicolon-separated NAME=REGEX pairs → [REDACTED:NAME] AICTL_SERVER_REDACTION_ALLOW= # semicolon-separated allowlist regexes; matches survive Layer-A/B AICTL_SERVER_REDACTION_NER=false # enable Layer-C NER (requires `redaction-ner` feature + pulled model) AICTL_SERVER_REDACTION_NER_MODEL=onnx-community/gliner_small-v2.1 # NER model spec (owner/repo)
// security

Defense in depth

The same redaction and prompt-injection controls the CLI ships run on every gateway request — same regex bank, same entropy pass, same optional NER, same config.

Master-key gate

Constant-time bearer-token compare. Both wrong-token and missing-token surface as the same 401 body so the distinction can't be enumerated.

Outbound redaction

aictl_core::run::redact_outbound runs on every gateway request. Local providers (Ollama / GGUF / MLX) skip unless AICTL_SECURITY_REDACTION_LOCAL=true.

Prompt-injection guard

Every user message passes through security::detect_prompt_injection. Matches get rejected with 400 prompt_injection so poisoned prompts can't burn the operator's tokens.

Server log & audit

Two sinks share one event source: JSON-Lines appended to ~/.aictl/server.log (override with AICTL_SERVER_LOG_FILE, empty disables) and human-readable lines on stderr. Level via AICTL_SERVER_LOG_LEVEL; rotation is the operator's job.

A separate audit trail records every successful gateway dispatch as gateway:<provider> (tagged with the per-request UUID) and every redaction event to a per-process JSON-Lines file at ~/.aictl/server-audit.log (override with AICTL_SERVER_AUDIT_FILE or --audit-file; empty disables). Toggle the whole stream with AICTL_SERVER_SECURITY_AUDIT_LOG. Unlike the CLI's session-keyed scheme, the server uses one file for the whole process — there is no session id.

Rate limit (optional)

Per-client-IP token bucket configurable via AICTL_SERVER_RATE_LIMIT_RPM and AICTL_SERVER_RATE_LIMIT_BURST. Off by default. Saturation returns 429 with a Retry-After header. The global concurrency cap (AICTL_SERVER_MAX_CONCURRENT_REQUESTS) operates independently and remains the primary backpressure mechanism.

Concurrency cap

A global semaphore caps in-flight requests at AICTL_SERVER_MAX_CONCURRENT_REQUESTS (default 32). Saturation returns 503 immediately rather than queueing — clients that care about backpressure handle retry themselves.
