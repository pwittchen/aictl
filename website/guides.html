// 01 getting started

Install and run

One command installs the prebuilt binary. No runtime, no daemon, no account.

01

Install

$ curl -sSf https://aictl.app/install.sh | sh

Downloads the binary for your platform into ~/.local/bin/aictl. Set AICTL_INSTALL_DIR to pick a different location.

02

Configure

$ aictl --config

Interactive wizard. Pick a provider, pick a model, paste an API key. Re-runnable at any time.

03

Start chatting

$ aictl

Launches the interactive REPL. Type a message, or use a slash command like /help.

// 02 configuration

Configuration

All state lives under ~/.aictl/. The single config file is ~/.aictl/config — plain KEY=VALUE lines, comments with #, quoted values and export prefixes both accepted.

Via the wizard

Run aictl --config for an interactive prompt that writes provider, model, and API keys to ~/.aictl/config (or your keyring). The same wizard is available inside the REPL as /config.

By hand

Open ~/.aictl/config in any editor. CLI flags always override config values, so you can try a different provider with --provider without touching the file.

Core keys

  • AICTL_PROVIDER — default provider (e.g. anthropic, openai, gemini, grok, mistral, deepseek, kimi, zai, ollama, gguf, mlx)
  • AICTL_MODEL — default model name
  • AICTL_MEMORY — long-term (all messages, default) or short-term (last 20-message sliding window)
  • AICTL_INCOGNITO — true to start the REPL without saving sessions (default false)
  • AICTL_PROMPT_FILE — project prompt filename loaded from the current directory (default AICTL.md)
  • AICTL_PROMPT_FALLBACK — when the primary prompt file is missing, fall back to CLAUDE.md then AGENTS.md (default true)
  • AICTL_TOOLS_ENABLED — false disables every tool; the LLM answers in plain text only (default true)
  • AICTL_AUTO_COMPACT_THRESHOLD — context usage % at which the REPL auto-compacts (1–100, default 80)
  • AICTL_LLM_TIMEOUT — per-call LLM timeout in seconds; 0 disables (default 30)
  • AICTL_MAX_ITERATIONS — max LLM calls per agent turn (default 20)

A project-specific prompt can live in an AICTL.md file in the current working directory. When present, its contents are appended to the system prompt — useful for per-project instructions.

Example ~/.aictl/config:

AICTL_PROVIDER=anthropic AICTL_MODEL=claude-sonnet-4-20250514 LLM_ANTHROPIC_API_KEY=sk-ant-...

API keys

Only the keys matching your active provider are required. Add more if you plan to switch providers mid-session. Ollama, GGUF and MLX run locally and need no key.

Required (per provider)

  • LLM_OPENAI_API_KEY — OpenAI
  • LLM_ANTHROPIC_API_KEY — Anthropic
  • LLM_GEMINI_API_KEY — Google Gemini
  • LLM_GROK_API_KEY — xAI Grok
  • LLM_MISTRAL_API_KEY — Mistral
  • LLM_DEEPSEEK_API_KEY — DeepSeek
  • LLM_KIMI_API_KEY — Kimi (Moonshot)
  • LLM_ZAI_API_KEY — Z.ai
  • LLM_OLLAMA_HOST — Ollama URL (default http://localhost:11434)
  • FIRECRAWL_API_KEY — optional, enables search_web

Where to get them

  • OpenAI — platform.openai.com/api-keys
  • Anthropic — console.anthropic.com
  • Gemini — aistudio.google.com
  • Grok — console.x.ai
  • Mistral — console.mistral.ai
  • DeepSeek — platform.deepseek.com
  • Kimi — platform.moonshot.ai
  • Z.ai — z.ai
  • Ollama — ollama.com/download (no key needed)
  • Firecrawl — firecrawl.dev
// 04 secure storage

Secure key storage

By default, API keys sit as plain text in ~/.aictl/config. aictl can also store them in the OS-native keyring — macOS Keychain or Linux Secret Service — and reads them transparently from whichever store has them.

Lock

Copies every plain-text key from ~/.aictl/config into the keyring and removes the plain-text copy. Via /keys in the REPL or --lock-keys at the CLI.

Unlock

Copies keyring entries back into ~/.aictl/config. Useful when moving to a machine without a keyring daemon. Via /keys or --unlock-keys.

Clear

Removes keys from both stores (asks for confirmation). Via /keys
or --clear-keys.

The welcome banner shows the active backend, e.g. keys: Keychain (2 locked · 1 plain · 0 both). When no keyring is available the banner reads keys: plain text and aictl falls back transparently.

// 05 providers

Providers & models

Eight remote APIs plus three local backends. Switch with /model inside the REPL — the conversation context survives the switch.

OpenAI

GPT-4.1 family, GPT-4o, GPT-5 / 5.2 / 5.4, o1 / o3 / o4-mini. Vision on every model. Image generation via DALL-E 3.

Anthropic

Claude Haiku, Sonnet, and Opus — 4.x, 4.1, 4.5, 4.6, 4.7. Vision on every model. Prompt caching is explicit and active.

Google Gemini

Gemini 3.1 Pro / 3 Flash / 3.1 Flash Lite (preview) and 2.5 Pro / Flash / Flash Lite. Vision on every model. Image generation via Imagen 4.0.

xAI Grok

grok-4.20 and grok-4, grok-4-fast / 4.1-fast (2M-token context), grok-3 / mini. Vision on every model. Image generation via Grok 2 Image.

Mistral

mistral-large, medium, small, codestral. Vision on every model. No image generation.

DeepSeek

deepseek-chat and deepseek-reasoner. Text only.

Kimi (Moonshot)

kimi-k2.6 / k2.6-thinking, kimi-k2.5, and moonshot-v1-8k / 32k / 128k. Vision on the k2.6 / k2.5 / moonshot-v1 variants.

Z.ai

glm-5 / 5-turbo / 5.1, glm-4.7 / 4.7-flashx / 4.7-flash, glm-4.6, the glm-4.5 family (base / x / airx / air / flash), and glm-4-32b-0414-128k. Text only.

Ollama (local)

Any model pulled via ollama pull. Discovered automatically from the local server. Vision depends on the model (e.g. llava, llama3.2-vision).

Native GGUF (local)

GGUF models via llama.cpp, in-process — no Ollama server required. Managed under /gguf. Experimental.

Native MLX (local)

MLX 4-bit models on Apple Silicon, in-process — no Python. Managed under /mlx. Experimental. The macOS aarch64 release ships with mlx.metallib placed next to the binary at install time; other platform releases contain only the aictl binary.

Cost meter

After every turn, aictl prints tokens, estimated cost, and wall time. View daily and monthly totals with /stats.

Per-model pricing lives in the project README under Providers. For full pricing details, visit the providers' websites. Local providers always show $0.00.

// 06 modes

Usage modes

Two ways to talk to the agent: interactive REPL with persistent conversation history, or one-shot for scripting and pipelines.

Interactive REPL

Run aictl with no --message to enter the REPL. Full history, slash commands, markdown rendering, and sessions that persist to disk. Press Esc to interrupt a running LLM call or tool execution — the turn is rolled back.

Single-shot

Pass --message "..." to run a one-off prompt and exit. Combine with --auto to skip tool-confirmation prompts, or --auto --quiet to print only the final answer (ideal for scripts).

Autonomous mode

--auto skips the y/N confirmation before every tool call — the agent runs to completion without asking. Use with care inside directories that matter.

Unrestricted mode

--unrestricted disables every security check (CWD jail, blocked commands, injection guard). Only use in fully trusted contexts.

// 07 repl

REPL commands

Inside the REPL, type / plus a command name. Press Esc at any time to interrupt an LLM call or a running tool.

Session & context

  • /clear — clear conversation context
  • /compact — summarize the conversation
  • /context — show tokens & messages vs. limits
  • /memory — switch long-term / short-term memory
  • /copy — copy last response to clipboard
  • /session — show, name, browse, or clear sessions
  • /stats — view today / month / overall usage
  • /exit — leave the REPL

Behavior & setup

  • /info — provider, model, version, OS
  • /model — switch provider and model live
  • /behavior — toggle auto / human-in-the-loop
  • /agent — manage agents (create, load, delete)
  • /skills — manage one-turn skills (create, invoke, delete)
  • /tools — list available tools
  • /ping — validate API keys and probe provider connectivity
  • /security — show current security policy
  • /keys — lock / unlock / clear API keys
  • /config — re-run the config wizard

Local models

  • /gguf — view / pull / remove GGUF models
  • /mlx — view / pull / remove MLX models (Apple Silicon)

Version & updates

  • /version — check for a newer release
  • /update — update to the latest version
  • /uninstall — remove the aictl binary
  • /help — show the command list
// 08 cli flags

CLI flags

Long-form only by convention. The only short flags are -v (--version)
and -h (--help).

Core

  • --message — one-shot prompt (omit for REPL)
  • --provider — override provider
  • --model — override model
  • --auto — skip tool confirmation
  • --quiet — final answer only (requires --auto)
  • --unrestricted — disable security checks
  • --incognito — no session save
  • --config — interactive wizard
  • --version, -v — print version
  • --help, -h — print help

Agents, skills & sessions

  • --agent <name> — load a saved agent
  • --list-agents — print saved agents and exit (add --category <name> to filter)
  • --pull-agent <name> — pull an official agent from the aictl repo (add --force to overwrite)
  • --skill <name> — invoke a saved skill for one turn
  • --list-skills — print saved skills and exit
  • --session <id|name> — resume a session
  • --list-sessions — print saved sessions and exit
  • --clear-sessions — delete all saved sessions

API keys

  • --lock-keys — move plain keys into the keyring
  • --unlock-keys — move keyring keys to config
  • --clear-keys — delete keys from both stores

Maintenance

  • --update — update to latest version
  • --uninstall — remove the binary (leaves ~/.aictl/)

Local models

  • --pull-gguf-model <spec>
  • --list-gguf-models · --remove-gguf-model <name> · --clear-gguf-models
  • --pull-mlx-model <spec>
  • --list-mlx-models · --remove-mlx-model <name> · --clear-mlx-models

Redaction NER models

  • --pull-ner-model <spec> — e.g. onnx-community/gliner_small-v2.1
  • --list-ner-models · --remove-ner-model <name> · --clear-ner-models

CLI flags always take priority over values in ~/.aictl/config.

// 09 agents

Agents

Agents are reusable system-prompt extensions that specialize the LLM for a dedicated task or persona. They live as plain text under ~/.aictl/agents/.

Create

Use /agent in the REPL. Pick Create manually to paste your own prompt, or Create with AI to have the current model draft one from a short description.

Browse official

Pick Browse official agents to pull curated agents (e.g. researcher, software-architect, critic) from the aictl repo. Agents are grouped by category; rows show [ ], [✓], or [↑] to reflect pull state. Non-interactive equivalent: --pull-agent <name> (add --force to overwrite).

Load

Via /agent → View all, or pass --agent <name> on the command line. Works in both single-shot and REPL modes. Agents pulled from the catalogue carry an [official] badge in the list.

Naming

Agent names may contain letters, numbers, underscores, and dashes only. When loaded, the agent name appears in magenta brackets on the REPL prompt: [my-agent] ❯.

Unload & delete

Pick Unload in the /agent menu to drop the active agent mid-session, or Delete to remove a saved file. --list-agents prints every saved agent.

// 10 skills

Skills

Skills are markdown playbooks invoked on demand for a single turn — unlike agents, which persist for the whole session. A skill encodes a repeatable procedure (“run the commit workflow”, “review the pending diff”) that vanishes once the turn completes. Skills live under ~/.aictl/skills/<name>/SKILL.md; override the location with AICTL_SKILLS_DIR.

Create

Open /skills in the REPL. Create manually asks for a name, a one-line description, and the markdown body. Create with AI has the current model draft the body from a short description.

Invoke

Type /<skill-name> to run the skill with a default trigger, or /<skill-name> <task> to pass a task as the user message. Outside the REPL, use --skill <name> — works with or without --message.

Format

Each SKILL.md starts with YAML frontmatter (name, description) followed by the markdown procedure. Unknown frontmatter fields are ignored so forward-compatible additions don’t break older clients.

Scope & safety

The body is merged into the base system prompt for one turn only and is never saved into session history. Names collide-checked against built-in slash commands (help, exit, agent, …) at save time so a skill can’t shadow a command. --list-skills prints every saved skill.

// 11 sessions

Sessions

Every REPL run is a session. A new UUID is generated at startup and the conversation is persisted after every turn under ~/.aictl/sessions/.

Name & resume

Use /session → Set name to give a run a human-readable alias. Resume later
with --session <uuid|name>.

Browse & clear

List saved sessions with --list-sessions, delete one from inside the /session menu, or wipe everything with --clear-sessions.

Incognito

--incognito (or AICTL_INCOGNITO=true) disables session creation entirely. A yellow notice appears in the banner and /session reports that it is disabled.

// 31 tools

Tools the agent can use

Every tool call is validated by the security policy and — unless --auto is set — confirmed with a y/N prompt before running.

Files & shell

  • exec_shell — run a shell command
  • read_file · write_file · edit_file
  • remove_file · create_directory
  • list_directory · find_files
  • search_files — grep across the tree
  • diff_files — unified diff, 3 lines of context, in-process
  • archive — create, extract, list tar.gz / tgz / tar / zip
  • checksum — streaming SHA-256 / MD5 digests
  • git — restricted status / diff / log / blame / commit

Web & media

  • fetch_url · extract_website
  • search_web — Firecrawl-backed
  • fetch_datetime · fetch_geolocation
  • read_image — vision on supported providers
  • generate_image — DALL-E, Imagen, Grok
  • read_document — PDF, DOCX, spreadsheets

Code & data

  • run_code — python, node, ruby, perl, lua, bash
  • lint_file — auto-selected linter per extension
  • json_query — jq-style JSON filters
  • csv_query — SQL-like over CSV/TSV
  • calculate — safe math expression evaluator

Host & network

  • list_processes — structured `ps` with filters (name/user/pid/%cpu/%mem/port)
  • check_port — TCP reachability probe (host:port, DNS + URL forms)
  • system_info — OS, CPU, memory, disk info (macOS + Linux)
  • clipboard — read / write the system clipboard (pbcopy, wl-copy, xclip)
  • notify — desktop notifications (osascript / notify-send)

All tool output is sanitized to prevent prompt injection via <tool> tags. Shell subprocesses run with a scrubbed environment (API keys stripped) and a 30-second timeout by default.

// 12 security

Security

aictl is locked down by default. Review the active policy any time with /security.

CWD jail

File tools only operate within the current working directory. Path traversal via .. is blocked via canonicalization.

Blocked paths

Sensitive paths are always refused: ~/.ssh, ~/.gnupg, ~/.aws, ~/.config/gcloud, ~/.aictl, /etc/shadow, /etc/sudoers.

Shell allowlist

Dangerous commands (rm, sudo, dd, mkfs, nc, …) are blocked. Command substitution ($(), backticks) is rejected. Compound commands are split and each segment validated.

Env scrubbing

Shell subprocesses see no environment variable matching *_KEY, *_SECRET, *_TOKEN, or *_PASSWORD. Your API keys cannot leak into a tool call.

Injection guard

User prompts containing instruction-override phrases or forged role / tool tags are blocked before reaching the LLM. Disable with AICTL_SECURITY_INJECTION_GUARD=false.

Audit log

Every tool call appends one JSON line to ~/.aictl/audit/<session-id> — timestamp, tool, input, outcome. Observability-only, so --unrestricted leaves it on. Disable with AICTL_SECURITY_AUDIT_LOG=false.

Sensitive-data redaction

Outbound messages can be screened at the network boundary for credentials (API keys, AWS / GitHub / Stripe tokens, JWTs, PEM private keys, DB connection strings) and PII (emails, phones, credit cards via Luhn, IBANs via mod-97, high-entropy tokens). Enable with AICTL_SECURITY_REDACTION=redact to swap each hit for [REDACTED:<KIND>], or =block to abort the turn. Off by default. Optional NER pass (person / location / organization) via the redaction-ner cargo feature.

Customization

Every setting is configurable via AICTL_SECURITY_* keys. Disable individual tools with AICTL_SECURITY_DISABLED_TOOLS. Bypass everything with --unrestricted.

Review in REPL

Run /security inside the interactive REPL to print the live policy — blocked commands, CWD jail status, timeouts, env scrubbing, injection guard, audit log, redaction mode with its detector / custom pattern / allowlist / NER breakdown, and per-key storage locations. Useful to confirm a config change actually took effect.

Security config keys

  • AICTL_SECURITY — master security switch (default true)
  • AICTL_SECURITY_INJECTION_GUARD — block prompt-injection attempts (default true)
  • AICTL_SECURITY_CWD_RESTRICT — restrict file tools to the working directory (default true)
  • AICTL_SECURITY_SHELL_ALLOWED — comma-separated whitelist of allowed shell commands (empty = all except blocked); e.g. ls,cat,grep,find
  • AICTL_SECURITY_SHELL_BLOCKED — comma-separated list of extra blocked shell commands, added to built-in defaults; e.g. curl,wget,ssh
  • AICTL_SECURITY_BLOCK_SUBSHELL — block $(), backticks, and process substitution (default true)
  • AICTL_SECURITY_BLOCKED_PATHS — comma-separated list of extra blocked file paths, added to built-in defaults; e.g. /etc/passwd,~/.bashrc,~/secrets
  • AICTL_SECURITY_ALLOWED_PATHS — comma-separated list of paths allowed outside the working directory; e.g. /tmp,~/Downloads,/var/log
  • AICTL_SECURITY_SHELL_TIMEOUT — shell command timeout in seconds (default 30)
  • AICTL_SECURITY_MAX_WRITE — max file write size in bytes (default 1048576 = 1 MB)
  • AICTL_SECURITY_DISABLED_TOOLS — comma-separated list of tool names to disable; e.g. exec_shell,search_web
  • AICTL_SECURITY_BLOCKED_ENV — comma-separated list of extra env vars to scrub from shell subprocesses; e.g. GITHUB_TOKEN,AWS_SESSION_TOKEN
  • AICTL_SECURITY_AUDIT_LOG — log tool calls and redaction events to ~/.aictl/audit/ (default true)

Redaction config keys

  • AICTL_SECURITY_REDACTION — outbound redaction mode: off (default), redact, or block
  • AICTL_SECURITY_REDACTION_LOCAL — also redact when sending to Ollama / GGUF / MLX (default false — local traffic never leaves the machine)
  • AICTL_REDACTION_DETECTORS — comma-separated subset of built-ins (empty = all): api_key, aws, jwt, private_key, connection_string, credit_card, iban, email, phone, high_entropy
  • AICTL_REDACTION_EXTRA_PATTERNS — semicolon-separated NAME=REGEX pairs; each match becomes [REDACTED:NAME]. Example: CUSTOMER_ID=CUST-\d{8};TICKET=JIRA-\d{4,}
  • AICTL_REDACTION_ALLOW — semicolon-separated regexes whose matches suppress any overlapping detection (e.g. documentation examples or known-good hashes)
  • AICTL_REDACTION_NER — enable optional Layer-C NER (person / location / organization). Requires the redaction-ner cargo feature and a pulled model (default false)
  • AICTL_REDACTION_NER_MODEL — NER model spec owner/repo (default onnx-community/gliner_small-v2.1)

Pull an NER model (management works on every build):

$ aictl --pull-ner-model onnx-community/gliner_small-v2.1

Manage existing models:

$ aictl --list-ner-models
$ aictl --remove-ner-model <name>
$ aictl --clear-ner-models

Running inference requires building with the redaction-ner cargo feature:

$ cargo build --features redaction-ner
// 13 local models

Local models

Run models on your own hardware — no API key, no remote call, no per-token cost. Three options: Ollama, native GGUF, and native MLX.
Please note, GGUF and MLX are experimental and may have bugs.

Ollama

Install from ollama.com, then ollama pull <model> and ollama serve. aictl discovers pulled models automatically via the REST API. Set AICTL_PROVIDER=ollama.

Native GGUF

In-process llama.cpp — no Ollama required. Pull with aictl --pull-gguf-model hf:owner/repo/file.gguf or use the /gguf menu. Experimental; small models may struggle with tool-call formatting.

Native MLX

Apple Silicon only. Quantized 4-bit models from the mlx-community HF org. Pull with aictl --pull-mlx-model mlx-community/Llama-3.2-3B-Instruct-4bit or use the /mlx menu. Experimental.

Prebuilt binaries from the install script ship with GGUF support on every platform. The macOS Apple Silicon release additionally ships with --features mlx enabled and includes a sibling mlx.metallib next to the binary (MLX needs the Metal library at runtime); every other platform's release contains just the aictl binary. To rebuild from source with both features enabled:

$ cargo install --path . --features "gguf mlx"

// 15 update & uninstall

Update & uninstall

Keep aictl current, or remove it cleanly when you are done.

Update

Run /update in the REPL or aictl --update from the shell. Both pull the latest release and swap the binary in place.

Uninstall

Run aictl --uninstall (or /uninstall in the REPL) to remove the binary from ~/.local/bin/ and ~/.cargo/bin/. Your config, agents, and sessions under ~/.aictl/ are left intact.

Wipe state

To also remove config, API keys, saved agents, and sessions, delete ~/.aictl/ manually: rm -rf ~/.aictl.
